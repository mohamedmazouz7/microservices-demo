┌─────────────┐
│  FRONTEND   │  Role: User Interface
│  (Nginx)    │  ├─ Serves HTML/CSS/JS
└──────┬──────┘  ├─ Handles user login
       │         ├─ Displays products
       │         └─ Shows user interface
       │
       │ Calls:
       ├─► POST /auth/login                    → Get JWT Token
       ├─► GET  /products                      → Fetch product list
       ├─► POST /products (with JWT)           → Add product
       ├─► GET  /users (with JWT)              → Fetch user list
       └─► POST /users (with JWT)              → Add user


┌─────────────────┐
│ AUTH SERVICE    │  Role: Token Manager (Stateless)
│  (FastAPI)      │  ├─ Issues JWT tokens
└────────┬────────┘  ├─ Validates JWT tokens
         │           ├─ Uses HS256 signature
         │           └─ NO database
         │
         │ Endpoints:
         ├─► POST /auth/login                  → Issue token
         ├─► POST /auth/verify                 → Verify token
         ├─► GET  /healthz                     → Health check
         └─► GET  /ready                       → Readiness check
         │
         │ Called by:
         ├─ Frontend (login)
         ├─ Product Service (token verification)
         └─ User Service (token verification)


┌──────────────────────┐
│  PRODUCT SERVICE     │  Role: Product Catalog Manager
│   (FastAPI)          │  ├─ Manage products
└────────┬─────────────┘  ├─ Require auth for write
         │                ├─ Allow public read
         │                └─ Verify tokens with Auth
         │
         │ Endpoints:
         ├─► GET  /products              → List all products (public)
         ├─► POST /products (+ JWT)      → Add product (protected)
         ├─► GET  /healthz               → Health check
         ├─► GET  /ready                 → Readiness check
         └─► GET  /db-check              → DB connection check
         │
         │ Dependencies:
         ├─ Product DB (PostgreSQL)      ← Query products
         ├─ Auth Service                 ← Verify JWT tokens
         │
         │ Called by:
         └─ Frontend (via Ingress)


┌──────────────────────┐
│   USER SERVICE       │  Role: User Manager
│   (FastAPI)          │  ├─ Manage users
└────────┬─────────────┘  ├─ Require auth
         │                ├─ Verify tokens
         │                └─ Persist to DB
         │
         │ Endpoints:
         ├─► GET  /users                 → List all users (protected)
         ├─► POST /users (+ JWT)         → Create user (protected)
         ├─► GET  /users/{id} (+ JWT)    → Get user by ID (protected)
         ├─► GET  /healthz               → Health check
         ├─► GET  /ready                 → Readiness check
         └─► GET  /db-check              → DB connection check
         │
         │ Dependencies:
         ├─ User DB (PostgreSQL)         ← Persist user data
         ├─ Auth Service                 ← Verify JWT tokens
         │
         │ Called by:
         └─ Frontend (via Ingress)

╔════════════════════════════════════════════════════════════════════════════════╗
║                       AUTHENTICATION FLOW DETAIL                                ║
╚════════════════════════════════════════════════════════════════════════════════╝

1. USER SUBMITS LOGIN FORM
   ├─ Username: testuser
   └─ Password: testpass

2. FRONTEND CALLS AUTH SERVICE
   ├─ POST /auth/login
   ├─ Body: {"username": "testuser", "password": "testpass"}
   └─ No Authorization header needed

3. AUTH SERVICE PROCESSES REQUEST
   ├─ Validates credentials against hardcoded list
   ├─ Creates JWT token
   │  ├─ Header: {alg: HS256, typ: JWT}
   │  ├─ Payload: {sub: testuser, exp: <timestamp>}
   │  └─ Signature: HMACSHA256(header+payload, secret)
   └─ Returns: {access_token: "eyJ...", token_type: "bearer"}

4. FRONTEND STORES TOKEN
   ├─ Saves in localStorage
   └─ Subsequent requests include: Authorization: Bearer <token>

5. ACCESSING PROTECTED ENDPOINTS
   ├─ Frontend includes token in Authorization header
   ├─ Service receives request
   ├─ Service extracts token from header
   ├─ Service calls Auth Service: POST /auth/verify {token}
   ├─ Auth Service validates:
   │  ├─ Checks signature matches (using secret)
   │  ├─ Checks token not expired
   │  └─ Returns 200 if valid, 401 if invalid
   └─ Service allows/denies based on response

6. VALID TOKEN SCENARIO
   ├─ Auth Service: ✓ Signature valid
   ├─ Auth Service: ✓ Not expired
   ├─ Auth Service returns: 200 OK
   └─ Service processes request + updates DB

7. INVALID TOKEN SCENARIO
   ├─ Auth Service: ✗ Signature invalid OR
   ├─ Auth Service: ✗ Token expired
   ├─ Auth Service returns: 401 Unauthorized
   └─ Service returns: 401 to client (no DB update)
